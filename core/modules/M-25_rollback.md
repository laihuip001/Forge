# M-25: Rollback (可逆性保証)

## 目的

すべての変更を元に戻せる状態を維持する。

## ルール

1. **ファイル変更前**にバックアップを作成する（または Git コミット）
2. **破壊的操作**（削除、上書き）の前に確認を求める
3. **ロールバック手順**を常に用意する

## 実装パターン

| 操作 | ロールバック手段 |
|---|---|
| ファイル編集 | Git での `revert` |
| ファイル削除 | 削除前に `archive/` へ移動 |
| ディレクトリ変更 | 変更前の構造をログに記録 |

## 禁止事項

- `rm -rf` を確認なしで実行しない
- Git 履歴を強制的に書き換えない (`--force` 禁止)
